\begin{abstract}
\noindent
When a language model must choose one answer from a large space of equally valid options, a
format clause --- \emph{``Reply with JSON only''} --- changes which answer it chooses. We re-run
the One-Word Census \citep{parikh2026census}, 31 category prompts with wide answer spaces
asked of 44 models, with the reply requested in JSON: no schema enforcement, no constrained
decoding, only the request. The field's convergence deepens sharply: on the unconstrained
prompt (``Pick a word.''), the modal answer's share rises from 40\% to 63\% and distinct
answers fall from 53 to 28; mean answer-choice surprisal drops 1.80 to 1.58 bits. The tax is
\emph{progressive} and respects the instrument's resolution: seven of 44 models move
individually (BH-FDR $q=.10$), all toward the mode, led by the panel's two most distinctive
frontier models (the strongest explorer loses 1.32 bits --- half its measured
distinctiveness) while the conformist floor is unmoved; one family-level effect survives
aggregation --- the Llama-derived models (excepting a heavily third-party-retuned one)
\emph{diverge} under JSON ($+0.70$ bits against the rest of the field, $p<10^{-4}$) --- and
the remaining deltas form a noise plateau we decline to rank. The format acts as a sharpener, not a re-indexer --- the plain-chat modal answer
survives in 28 of 31 categories, and the three flips occur exactly where the plain mode was
weakest. The column also shows that a model's \emph{defaults are register-indexed}, read from discrete
behavior rather than score deltas: of the panel's 144 stable off-modal defaults (the same
non-modal answer, four runs of four, in chat), only 47\% survive into JSON. Retention is a
per-model trait --- GPT-5.6 Terra loses its signature defaults (\emph{mustard}$\to$ketchup,
\emph{mango}$\to$apple, four-of-four flips), while Claude Fable 5 retains most of its own
(5 of 7) and acquires \emph{new} stable defaults that exist only in the register
(\emph{cerulean}, \emph{carpenter}, four-of-four in JSON, never in chat). Personality is
indexed to the channel, as the census found answers indexed to wording. Full-battery controls reveal a register gradient: the compression is significant and specific
to the answer-delivery formats models are trained to speak (JSON $-0.22$ bits, $p=10^{-4}$;
XML $-0.18$, $p=.003$; sign-flip permutation over models, corroborated over categories),
absent for YAML and CSV ($p>.15$), and \emph{reversed} for an arbitrary non-data wrapper ---
square brackets significantly loosen the field ($+0.16$, $p=.002$). On the unconstrained
prompt, all four serializations concentrate the pool (serendipity 41\% plain $\to$ 52--65\%),
so a corpus-register component remains visible; but the battery-wide gradient weighs toward
tool-use post-training as the dominant mechanism. Structured output is how software consumes
language models; that surface is served by a measurably more homogeneous model than the chat
surface on which models are evaluated, compared, and chosen.
\end{abstract}

\section{Method}
\label{sec:method}

\subsection{Instrument}
% Census recap: 31 single-turn categories, no system prompt, temp 1.0, 4 runs; answer-choice
% surprisal, leave-one-out, add-one smoothed; exact match on normalized tokens; junk guard.
% CRITICAL design point: surprisal is computed WITHIN each format column (json answers scored
% against the json field), so a column's compression is a property of the column, not a
% cross-register comparison artifact.

\subsection{Format columns}
% json: full 31-category battery, all 44 models x 4 runs (5{,}456 calls)
% xml, brackets, yaml, csv: 6-category controls (color, fruit, condiment, cheese, tree,
% any_word), all 44 models x 4 runs each
% clauses verbatim:
%   json:     Reply with JSON only, in the form {"word": "<your answer>"}.
%   xml:      Reply with XML only, in the form <word>your answer</word>.
%   brackets: Reply with your answer inside square brackets only, like [answer].
%   yaml:     Reply with YAML only, in the form `word: <your answer>`.
%   csv:      Reply with CSV only: a header row `word`, then one data row with your answer.
% No response_format parameter, no tool schema, no constrained decoding anywhere.

\subsection{Parsing and hygiene}
% Format wrapper stripped by regex; extracted token passes the census junk guard; unwrapped
% replies fall back to the census rule directly (a malformed-JSON essay's last word must not
% read as a novel answer). Parse+junk survival >= 99\% per model; per-model survival audited
% for every model whose surprisal rose. Provenance: one interrupted run (credit exhaustion)
% refilled same-day; 54 of 8{,}624 json cells unrecoverable; stamped in the data file.

\subsection{The register moves the mode, not the temperature}
% Self-distinctness (distinct answers / runs; the census's effective-temperature proxy) is
% nearly FLAT at field level across columns: plain 0.42 | json 0.39 | xml 0.40 | yaml 0.42 |
% csv 0.43 | brackets 0.43 — while surprisal drops 0.22 bits. The collapse is POSITIONAL
% (mass relocates onto the field's mode), not a sampling change; rules out a serving-layer
% temperature artifact (if providers altered sampling on JSON-looking asks, selfd would
% crater everywhere). The within-model narrowing that exists is progressive, same tail:
% hermes 0.71->0.52, deepseek-v3.2 0.56->0.42, sonar 0.50->0.40, haiku-4.5 0.42->0.33;
% conformist floor flat (sonnet-5 0.30->0.27); fable flat 0.32 (conviction, not noise).
% => format tax is progressive on BOTH census axes; the two-axis dissociation (big surprisal
% move, small selfd move) is itself evidence for reshaped conditional distribution.
